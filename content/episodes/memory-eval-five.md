# 怎么评测记忆系统：五个方面

很多人一做 Agent 记忆，就满足于「它好像记得我叫什么」。评测要是只停在这，上线很容易翻车。更靠谱的方式，是至少从五个方面看。

第一，写入。该记的有没有写进去？会不会记错、记重、记进不该进的隐私？写入失败要有日志，不能假装成功。

第二，检索。到该用的时候，能不能把正确记忆召回来？召回太少，Agent 像失忆；召回太多噪声，又像被人塞小条干扰判断。

第三，更新与遗忘。用户改了偏好，旧记忆还在不在指挥行动？过期信息有没有衰减或删除？冲突出现时，谁说了算，要有规则。

第四，时序与一致性。先发生的事和后发生的事，能不能按时间线讲清楚？今天说「我素食」，明天却推荐烤肉，这就是一致性破产。

第五，端到端任务收益。记忆系统再漂亮，如果完成率、返工率、用户满意度没提升，那它只是成本。评测最终要落到业务指标，而不是只报几个中间分数。

补充一句现实：记忆强不强，很依赖场景和 Agent 的 harness，也就是外围脚手架怎么管状态、工具和流程。所以评测集最好贴近真实对话，别只用玩具问答。

你出门记住这五个词：写入、检索、更新、一致性、任务收益。面试官问「记忆怎么评」，按这条展开，就比空谈「用向量库存一下」扎实。
