# 大模型系统里的 Skill 是怎么加载的

最近大家都在说 Skill。你可以把它理解成：给 Agent 的一本「专项说明书」，里面写清楚这件事怎么做、能用哪些工具、边界在哪。它不是把所有话一次性糊进系统提示词。

为什么要单独加载？因为能力会越来越多。翻译、查库、画图、改代码，全塞进上下文，又贵又乱，模型还容易挑错工具。

常见加载方式大概三类。

第一，启动时加载核心 Skill。登录、权限、安全红线这些，几乎每次都要用，常驻更稳。

第二，按需加载。用户一开口，先做意图识别或检索，再把相关 Skill 拉进来。像工具箱，用到才打开那一层抽屉。

第三，分层加载。先给目录级摘要，让模型知道「有哪些能力」；真正执行时再展开细节。这样省上下文，也减少干扰。

面试里还容易被追问三件事。权限：这个 Skill 能不能碰生产数据。版本：升级后旧对话怎么兼容。冲突：两个 Skill 都说能处理同一问题，谁优先。

一句话记住：Skill 加载的本质，是在「能力完整性」和「上下文预算」之间做调度。加载对了，Agent 像懂行的同事；加载错了，就像把整间仓库倒在桌上，找不到锤子。
