# 把 Agent 源码读成一本书

有人说，把某个 Agent 项目的源码「写成了一本书」。这话听着玄，其实是一种很实用的学习法：别把仓库当一堆文件，把它当章节目录。

为什么有效？因为短视频和文档容易只给你名词：规划、工具调用、记忆、多 Agent。名词你会背，一落到代码路径就迷路。反过来，顺着源码走，你会看见真实的顺序：用户输入从哪进，怎么构造提示，何时决定调用工具，结果如何写回状态，失败怎么重试，何时结束循环。

你可以按「书」的结构拆任何 Agent 代码。

第一章，入口与会话。请求怎么进来，上下文如何组装。  
第二章，规划与决策。是固定工作流，还是模型自己选下一步。  
第三章，工具层。工具如何注册、如何鉴权、参数如何校验。  
第四章，记忆与状态。短对话、长记忆、外部存储各自放哪。  
第五章，观测与安全。日志、追踪、限流、人工确认点在哪。

读的时候做两件事就够。一是画一张呼叫关系草图，二是用自己的话复述「如果用户让它订票，代码会怎么走」。能复述，才算读懂。

这方法不绑某个具体仓库。换项目也能用。热点会过期，但「把复杂系统读成可讲的故事」这个能力不过期。

所以今天这条的重点不是安利某一个开源名，而是送你一个 Habit：遇到 Agent，先当书读，再当工具用。走路上把五章目录在心里过一遍，下次面试讲架构，你就不会只剩形容词了。
